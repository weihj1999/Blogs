7 Ways to Improve Public Cloud Security

"在你自己负责的数据中心上运行的应用是否安全？或者说是否可以信任云的安全性?"

最初，用户都会非常注重云中的数据安全性,仅仅几年之前，每个人都非常相信在数据中心
里面的应用确实更加安全, 但是我们尽力了一些变化，由商业软件公司Exacta和Pb7发起
的调研表明，其中一个促使SBMs选择迁移到公有云的一个原因是因为它提供的安全性。

非常类似的，Capital One的CIO， Rob Alexander，对云的安全性基于了很高的期望，他宣
布，Capital One押宝于Amazon Web Service，为他的移动银行应用提供的服务。

在很大程度上，这种变化在于云服务商自身， CSPs投入了很大的精力和重视到云的安全
性上,是因为他们知道如果没有一个方法来保证他们客户的数据的安全性，他们将没有任何
机会去谈商务。

因此，他们对最新的科技持续调研，研究网络攻击威胁，并投入响应的努力在保证他们系统
的安全性上。你甚至听说过，Apple公司，他们有自己的iCloud，正在和美国政府斗法，
为了避免在用户的系统上创建一个后门，因为这种行为无异于自杀。

最终，关于在哪里存放他们的数据的问题越来越少，但是他们采取了什么措施来避免来自于
安全犯罪行为的威胁。这里有一些重要的办法来提供你的公有云的安全性。

1. 在你的ALAs中提高云安全的重要级别

反复研读你的SLA，确认包含了安全承诺，记住如果在合同中不清楚，它就不能被执行
看起来需要确认的几个典型条目包含：
a. 我的数据是如何被保护的？加密策略是什么？
b. CSP会遵循什么样的安全标准？他们针对安全风险有什么样的安全防护？
c. 灾难恢复计划是什么？（DRP）
d. 我的云服务器活动是如何被监控和记录的？
e. 服务商如何处理服务到期的操作以及数据的删除？

2. 执行定期的策略检查和安全审计

云的使用情况，风险和使用机会统计。 大部分负责人没有任何形式的策略来确保在云中
的数据安全，这是一个值得关心的很大的漏洞。

你的公司应该花时间去创建一个定期的复检，更新你的安全策略信息。 进行定期的审计会
帮助你和你的云服务商紧密执行云安全策略和潜在的威胁。和CSP针对他们的安全执行和审
计，可能需要额外的权限

如果可能，你也可以要求第三方的安全审计服务， 这会确保你的安全流程目标清晰。

3. 遵从有效的监控和日志系统

你对于你能访问的所有环境的日志，是非常关键的，包括用户帐号密码的创建，删除，修改
，与其他系统的交互，同样非常重要。如果任何地方出了错误， 你就有一份记录，什么人
做了什么，什么时候一个关键操作步骤决定了云安全的漏洞。

4. 使用加密
加密可以使你保证进出于公有云的数据更安全，云服务商通畅给它们的客户提供自己的加密
服务，但是如果你的公司对于他们提供的安全级别不满意，，你也可以从第三方那里获取加
密服务。这种情况下，你和你的加密服务商才有权限拥有加密钥匙。

5. 在用户方面加强安全流程

一些调研表明，60%的安全问题是有公司内部的人的操作导致的，员工自身。 为了确保数据
真正得到保护，公司必须控制什么人访问什么样的数据。 你的CSP安全实践也需要在使用的
用户中得到执行，比如用户的安全协议，和很强的授权机制。

你也可能需要在用户名和密码之外包含其他的安全措施，比如说，物理令牌，安全卡，生物
识别等等。

6. 使用正确的安全工具和服务

如果你有和你的CSP的安全能力的工作经验和知识，那就用起来吧。 参考TSO／IEC 27000
的系列标准。 
你也可以使用工具来增强云的安全性，举例来说 BitGlass，允许你通过追踪和加密商务数
据来保护你的计算机和移动设备. SkyHigh 网络，另一方面，可以显示出来你的员工都使用
了哪些应用。

7. 测试漏洞
云的安全异味者你必须持续检查弱点，在每次你穿件一套系统的时候都要做这样的事情，其
实无论它在不在云上
一些CSP提供了弱点攻击测试的工具，比如说，Amzaon Inspector可以搜索风险和弱点，并
生成报告给你。

一些云需要共享责任，你和你的服务商必须在一起工作来确保你的公司和客户的数据得到保
保护，持续思考云的安全想就更重要的了
